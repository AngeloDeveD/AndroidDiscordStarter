# Исполняемая JVM-среда Discord ботов с поддержкой Lua-модулей 🚀

Это Android-приложение, позволяющее запускать полностью функционального Discord-бота прямо на мобильном устройстве в один клик. Бот поддерживает авторизацию по токену, мгновенную синхронизацию команд без перезапуска и динамическую обработку слэш-команд (Slash Commands, `/`), написанных на языке **Lua**.

---

## 🛠 Как устроена и работает программа

Приложение представляет собой легковесный автономный клиент с гибридной архитектурой:

1. **Автономное соединение (Discord Gateway API v10)**  
   Бот не требует внешних промежуточных серверов или обратных вызовов (Webhooks), для которых нужен SSL-сертификат или туннели (вроде ngrok). Соединение устанавливается напрямую между Android-устройством и серверами Discord по защищенным веб-сокетам (**WSS**). Пинг-таймеры (Heartbeats) отправляются в фоновом режиме для поддержания стабильности соединения.

2. **Движок JVM Lua (Luaj)**  
   Исполнение скриптов обеспечивается встроенным легковесным интерпретатором `Luaj-jse`. При запуске или нажатии кнопки синхронизации приложение сканирует назначенную папку на карте памяти устройства, считывает все файлы с расширением `.lua`, проверяет их синтаксис и автоматически регистрирует команды в системе Discord.

3. **Интерактивный терминал**  
   Все события — от процесса фазы авторизации (Identify) и удержания веб-сокетов, до ошибок в исполнении пользовательского Lua-кода (Lua Runtime errors) — транслируются в реальном времени с цветовым разделением логов в графитизированной консоли.

## 📂 Куда отправлять файлы скриптов `.lua`

При первом запуске программа автоматически инициализирует директорию на вашем устройстве и создает в ней четыре готовых примера (`ping.lua`, `calculator.lua`, `coinflip.lua` и `roll.lua`).

### Путь к папке скриптов на устройстве:
```bash
/sdcard/Android/data/com.aistudio.discordbot.lucqzx/files/commands/
```

### Способы передачи скриптов на Android:
1. **Через компьютер (USB-кабель)**  
   Подключите телефон к ПК в режиме передачи файлов (MTP). Перейдите по указанному выше пути во внутреннем накопителе и перебросьте свои файлы `.lua` в папку `commands`.
2. **Файловые менеджеры на телефоне**  
   Используйте продвинутые проводники с доступом к папкам Android-системы (например, **Solid Explorer**, **ZArchiver** или **Mi Проводник**).
3. **Графический редактор внутри приложения**  
   Во вкладке **Команды Lua** нажмите на иконку `+` (Создать), напишите или вставьте код, укажите имя и сохраните. Файл в директории создастся и загрузится автоматически!

## 📝 Руководство по написанию слэш-команд на Lua

Каждый файл в папке `commands/` должен представлять собой одну самостоятельную слэш-команду. Имя файла должно в точности совпадать с именем регистрируемой команды в Discord (например, файл `ban.lua` регистрирует команду `/ban`).

### Обязательная структура файла

Скрипт Lua должен содержать две глобальные структуры:
1. **Глобальный словарь `command`** — конфигурация команды для Discord.
2. **Глобальная функция `execute(interaction)`** — код, выполняющийся при вызове команды пользователем.

```lua
-- ШАБЛОН КОМАНДЫ (example.lua)
command = {
    name = "имя_команды",             -- Должно состоять строго из строчных латинских букв
    description = "Краткое описание",  -- Видно пользователям в Discord при вводе '/'
    options = {}                      -- Опции (аргументы) команды, если нужны
}

function execute(interaction)
    -- Логика выполнения команды
    return "Приветственный текст!"
end
```

---

### Спецификация таблицы `command`

Таблица `command` содержит конфигурационные метаданные.

#### Поле `options` (Аргументы команды):
Каждая опция внутри `options` — это таблица со следующими полями:
* `name` (string) — Имя аргумента (латиницей, строчными).
* `description` (string) — Описание для пользователя в Discord.
* `type` (integer) — Код типа данных согласно спецификации Discord:
  * **`3`** — `STRING` (Текст)
  * **`4`** — `INTEGER` (Целое число)
  * **`10`** — `NUMBER` (Дробное число)
  * **`5`** — `BOOLEAN` (Да/Нет)
* `required` (boolean) — Является ли аргумент обязательным для заполнения (`true` / `false`).
* `choices` (table) — Полу-статический список вариантов выбора (выпадающий список). Каждая запись содержит `{ name = "Отображаемое имя", value = "Значение" }`.

---

### Спецификация аргумента `interaction` (Входной контекст)

Когда пользователь выполняет слэш-команду, движок Lua вызывает вашу функцию `execute(interaction)` и передает в нее готовую таблицу со следующими доступными полями:

| Поле в Lua | Тип данных | Описание |
| :--- | :--- | :--- |
| `interaction.user` | `string` | Имя пользователя в Discord, вызвавшего команду (напр., `johndoe`) |
| `interaction.userId` | `string` | Уникальный цифровой ID пользователя |
| `interaction.guildId` | `string` | ID сервера, где запущена команда (пусто, если команда вызвана в ЛС бота) |
| `interaction.channelId` | `string` | ID текстового канала |
| `interaction.options` | `table` | Таблица переданных пользователем значений. Ключами являются `name` аргументов |

---

### Спецификация возвращаемых значений из `execute`

Функция `execute` в Lua может возвращать ответ в двух разных форматах:

1. **Простой текстовый ответ (Обычная строка)**
   ```lua
   function execute(interaction)
       return "Команда выполнена успешно!"
   end
   ```
2. **Расширенный ответ (Словарь таблицы)**  
   Позволяет управлять приватностью сообщения (флаг `ephemeral` — сообщение видит только отправитель).
   ```lua
   function execute(interaction)
       return {
           content = "Эта надпись будет видна только вам!",
           ephemeral = true -- Светится только для отправителя
       }
   end
   ```

## 🎨 Примеры готовых команд

### Пример 1. Простая команда-пинг (`ping.lua`)
```lua
command = {
    name = "ping",
    description = "Проверить статус работы и пинг бота",
    options = {}
}

function execute(interaction)
    return "🏓 Понг! Бот находится на связи и стабильно запущен на Android!"
end
```

### Пример 2. Отправка конфиденциального эхо-сообщения (`say.lua`)
```lua
command = {
    name = "say",
    description = "Повторяет введенный вами текст конфиденциально",
    options = {
        {
            name = "text",
            description = "Введите сообщение",
            type = 3, -- STRING
            required = true
        }
    }
}

function execute(interaction)
    local text = interaction.options["text"]
    return {
        content = "Вы ввели скрытый текст: ||" .. text .. "||",
        ephemeral = true
    }
end
```

### Пример 3. Калькулятор с выпадающим списком операций (`calc.lua`)
```lua
command = {
    name = "calc",
    description = "Выполняет математическую операцию над двумя числами",
    options = {
        {
            name = "num1",
            description = "Первое число",
            type = 10, -- NUMBER (дробное)
            required = true
        },
        {
            name = "operation",
            description = "Выберите математическое действие",
            type = 3, -- STRING
            required = true,
            choices = {
                { name = "Сложить (+)", value = "+" },
                { name = "Вычесть (-)", value = "-" },
                { name = "Умножить (*)", value = "*" },
                { name = "Разделить (/)", value = "/" }
            }
        },
        {
            name = "num2",
            description = "Второе число",
            type = 10, -- NUMBER (дробное)
            required = true
        }
    }
}

function execute(interaction)
    local n1 = interaction.options["num1"]
    local op = interaction.options["operation"]
    local n2 = interaction.options["num2"]
    
    local result = 0
    if op == "+" then
        result = n1 + n2
    elseif op == "-" then
        result = n1 - n2
    elseif op == "*" then
        result = n1 * n2
    elseif op == "/" then
        if n2 == 0 then
            return { content = "❌ Невозможно разделить на ноль!", ephemeral = true }
        end
        result = n1 / n2
    end
    
    return "📊 **Калькуляция**\nВыражение: `" .. n1 .. " " .. op .. " " .. n2 .. "`\nРезультат: **" .. result .. "**"
end
```
